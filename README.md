# cyberwave-driver-skill

A [Claude Code](https://claude.ai/claude-code) skill that scaffolds a production-ready [Cyberwave](https://cyberwave.com) driver from scratch.

## What it does

When invoked, the skill asks you a few questions about your hardware and then scaffolds a complete, runnable driver project:

- `driver/main.py` — entrypoint, reads Edge Core environment variables
- `driver/driver.py` — twin sync loop, edge config handling
- `driver/hardware.py` — hardware abstraction stub ready for your SDK
- `Dockerfile` — multi-stage, production-ready image
- `docker-compose.yml` — local development setup
- `.env.example`, `.gitignore`, `.dockerignore`, `pyproject.toml`, `requirements.txt`, `LICENSE` (Apache 2.0), `README.md`

The skill drives the interactive conversation; the actual files are generated by `scaffold.py` — a stdlib-only Python script you can also run directly without Claude.

## Installation

```bash
# Global (available in all projects)
git clone https://github.com/cyberwave-os/driver-skill ~/.claude/skills/cyberwave-driver

# Or project-local
git clone https://github.com/cyberwave-os/driver-skill .claude/skills/cyberwave-driver
```

To update to the latest version:

```bash
cd ~/.claude/skills/cyberwave-driver && git pull
```

## Usage via Claude Code

In any Claude Code session, run:

```
/cyberwave-driver
```

Claude asks for your driver name, hardware description, author, SDK language (Python/C++), and whether you have child twins. It runs `scaffold.py` and then offers to help implement the hardware layer for your specific device.

## Usage without Claude (standalone)

```bash
# Interactive
python scaffold.py

# Non-interactive / CI
python scaffold.py \
  --name my-lidar-driver \
  --description "A SICK LiDAR over Ethernet using the SOPAS protocol" \
  --author "Acme Robotics" \
  --child-twins
```

Run `python scaffold.py --help` for all options.
